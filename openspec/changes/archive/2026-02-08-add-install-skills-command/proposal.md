# 変更提案: install-skills サブコマンドの追加

## 概要
前回の会話で整理した `agent-skills-rs install-skills` の挙動を、`xcom-rs` の CLI でも利用できるようにする。埋め込みスキルの発見とインストールを行う新しいサブコマンドを追加し、プロジェクト/グローバルの両スコープで操作できるようにする。

## 目的
- スキルの導入手順を `xcom-rs` 内に統合し、追加ツール無しでインストールできるようにする
- 非対話/JSON 出力に対応し、エージェント運用と自動化に適した UX を提供する

## スコープ
- `install-skills` サブコマンドの追加
- `skills/` 配下にスキル定義を同梱し、インストール元として利用
- 埋め込みスキルの探索とインストール（プロジェクト/グローバル）
- `--agent` / `--skill` / `--global` / `--yes` / `--non-interactive` / `--json` のサポート
- JSON での結果出力

## 非スコープ
- 外部リポジトリやネットワークからのスキル取得
- GUI や対話 UI の追加
- 既存の認証・課金フローの変更

## 前提・制約
- スキルはリポジトリ内の埋め込みディレクトリを起点に探索する
- 各スキル定義は `skills/<skill-name>/SKILL.md` に配置されていることを前提とする
- インストール先は `.agents/skills` を canonical とし、必要に応じてエージェント別ディレクトリへリンク/コピーする

## 依存関係
- 既存の CLI イントロスペクション（commands/schema/help）の更新
- ファイルシステム操作に関するテストユーティリティ

## リスク
- OS ごとのシンボリックリンク挙動差分
- プロジェクト/グローバルの解決規則が既存運用と衝突する可能性
