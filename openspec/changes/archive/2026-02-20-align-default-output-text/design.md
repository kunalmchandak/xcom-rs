# align-default-output-text Design

## 設計方針
- 既定出力は `text` を維持し、早期エラーでも一貫させる。
- `--output` が明示されている場合は、早期エラーでも指定形式で返す。
- 既存の通常実行経路の出力フォーマット解釈は変更しない。

## 構成案
- `Cli::try_parse()` 失敗時に、`std::env::args()` から `--output` を簡易解析して出力形式を推定する。
  - `--output <value>` と `--output=<value>` を認識
  - `--` 以降はオプション解析しない
  - 複数指定は最後を採用
  - 不正値は既定の `text` にフォールバック
- `--log-format` の検証は `output_format` 決定後に行い、早期エラーでも指定出力を尊重する。

## 代替案
- 早期エラーは常に JSON を維持する案
  - 既定 `text` の一貫性を損ない、ユーザー体験が不統一になるため採用しない。
