## Context

書き込み系コマンドはスタブであり、実API連携を行っていません。共通クライアント導入後に、各コマンドをHTTP呼び出しへ移行します。

## Goals / Non-Goals

**Goals:**
- 投稿作成/返信/スレッド、エンゲージメント、ブックマーク、メディアアップロードを実APIで実行する
- 429/401/403/5xx を共通エラー分類に統一する
- テストはネットワーク不要のモックで検証できるようにする

**Non-Goals:**
- CLIの引数/出力形式の変更
- 非同期化やストリーミング対応

## Decisions

- **投稿作成・返信**: `POST /2/tweets` を使用し、返信は `reply.in_reply_to_tweet_id` を設定する。
- **スレッド**: 1件目を投稿後、以降は直前のIDへ返信する形で逐次実行する。
- **ユーザーID解決**: いいね/RT/ブックマークで必要なユーザーIDは `GET /2/users/me` で取得する。
- **メディアアップロード**: `POST /2/media/upload` を利用し、レスポンスの `media_id` を返す。
- **モック検証**: `mockito` でHTTPリクエストのパス/メソッド/ボディを検証する。

## Risks / Trade-offs

- [スコープ不足] → 403時のエラーメッセージに必要スコープを明記する
- [スレッド部分失敗] → 既存の構造化エラーに作成済みIDを残す
- [メディアサイズ制限] → エラー時はAPIのメッセージを保持して返す
