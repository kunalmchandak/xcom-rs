# extract-cli-handlers Design

## 設計方針
- `main.rs` は引数解析・共通初期化・ルーティングに限定する。
- 具体的なコマンド実行はドメイン別ハンドラーで完結させる。
- 既存の出力形式と終了コードの互換性を最優先する。

## 構成案
- `src/handlers/mod.rs` が公開入口となり、ドメイン別に `handle_*` 関数を提供する。
- ハンドラーは `ExecutionContext`・`OutputFormat`・`AuthStore` 等の必要な依存だけを受け取る。

## 代替案
- `main.rs` のみで関数分割する案
  - 単一ファイルのままでは責務分離が曖昧で、構造的改善が限定的。

## 影響範囲
- `src/main.rs` とコマンド実行ロジック全体
- 既存テスト（必要に応じて更新）
