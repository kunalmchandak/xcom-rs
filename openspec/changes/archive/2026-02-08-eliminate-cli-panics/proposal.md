# eliminate-cli-panics Proposal

## 背景
本番コードに `unwrap` / `expect` が残っており、入力不正や初期化失敗時にパニック終了する可能性があります。CLI としては構造化エラーで終了する方が望ましいです。

## 目的
本番コードからパニック要因を排除し、ユーザーに一貫したエラー応答を返します。

## スコープ
- `src/main.rs` を中心に `unwrap` / `expect` を削除
- エラーは既存の `Envelope` 形式で返す

## 非スコープ
- テストコード内の `unwrap` 最適化
- 例外的なデバッグ用途の挙動変更

## 成果物
- パニックのない CLI 実行パス
- 失敗時の構造化レスポンス

## リスク/留意点
- 既存のテストがパニックを前提としている場合は更新が必要
