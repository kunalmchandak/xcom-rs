# 設計メモ

## 出力方針
- 既存のEnvelope構造に準拠する
- `data` に診断結果を格納し、`meta` は既存の `traceId` などを流用

## 収集項目（最小）
- 認証状態（authenticated / auth_mode / scopes）
- 認証ストレージのパス解決結果（ファイル保存の場合）
- 予算トラッカーのパス解決結果
- 非対話モード、dry-run、コスト/日次予算設定の有無

## エラー方針
- 診断情報の一部が取得不能でも、全体失敗にせず警告としてまとめる
- 完全に取得不能な場合のみ `error` を返し、nextSteps を提示する
