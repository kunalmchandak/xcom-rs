# add-shell-completions 提案

## なぜ
- サブコマンドが増え、手入力の負荷と誤入力が増える。
- 補完は人間オペの体験改善に直結する。

## 何を変えるか
- `xcom-rs completion --shell <bash|zsh|fish>` で補完スクリプトを生成する。
- READMEとdocs/examplesに導入手順を追加する。

### 対象範囲
- CLIサブコマンド追加
- 補完スクリプト生成
- ドキュメント更新

### 非対象
- 既存コマンドの引数仕様変更

### 成果物
- 補完出力コマンド
- 導入ガイド

### 受け入れ条件
- 3種類のシェル補完が生成できる
- READMEに導入手順が記載される

### リスク・留意点
- 生成方式はCLI定義と同期が必要
