# align-default-output-text Proposal

## 背景
`--output` 未指定時は `text` が既定ですが、起動直後のエラー経路では JSON 固定の出力になり、デフォルトの一貫性が崩れています。

## 目的
早期エラー（引数解釈失敗や `--log-format` 不正など）でも、`--output` の既定値と指定値を尊重し、出力形式の挙動を統一します。

## スコープ
- `--output` 未指定時の早期エラー出力を `text` へ統一
- `--output` 明示時は早期エラーでも指定形式を尊重
- 既存のコマンド成功/失敗出力の挙動は維持

## 非スコープ
- 出力フォーマット自体の追加・変更
- エラー内容（コードやメッセージ）の仕様変更
- ログ出力（`--log-format`）の形式変更

## 成果物
- 早期エラー出力の既定 `text` 化
- `--output` 明示時の早期エラー出力の整合
- 対応するテストの更新

## リスク/留意点
- 既存テストがデフォルトで JSON を前提にしている場合は修正が必要
- 早期エラー時の出力先（stdout/stderr）を変更しないこと
