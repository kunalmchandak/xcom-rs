# extract-cli-handlers Proposal

## 背景
`src/main.rs` の `main()` が巨大で、コマンドごとの処理・エラーハンドリング・出力整形が集中しています。変更の影響範囲が広くなり、テストやレビューの負担が増大しています。

## 目的
CLI コマンドの実行ロジックをドメイン別ハンドラーに分離し、`main.rs` を薄いルーティング層にすることで保守性とテスタビリティを高めます。

## スコープ
- コマンド実行ロジックを `src/handlers/` に抽出
- `main.rs` は引数解析・実行ルーティング・共通初期化のみを担当
- 既存の出力フォーマットと終了コードの挙動は維持

## 非スコープ
- 新しい機能追加や CLI 仕様変更
- 出力 JSON/YAML 形式の変更

## 成果物
- `src/handlers/` モジュール群
- `main.rs` の簡素化
- 既存テストの更新（必要に応じて）

## リスク/留意点
- ハンドラー分割時のルーティング漏れ
- 既存のエラーハンドリングの取りこぼし
