# refactor-cli-parse-tests 提案

## なぜ
- `src/cli.rs` に大量の同型テストが並んでおり、追加・修正時のコストが高い。
- テストの重複が多く、差分レビューがしづらい。

## 何を変えるか
- CLI パースのテストをテーブル駆動にまとめ、重複を削減する。
- テストを専用モジュールへ分離し、CLI 定義の読みやすさを改善する。

### 対象範囲
- CLI パーステストの再構成
- テスト用ヘルパーの導入

### 非対象
- CLI の引数仕様変更
- 実装の動作変更

### 成果物
- テーブル駆動の CLI パーステスト
- 簡素化された `src/cli.rs`

### 受け入れ条件
- `cargo test cli::tests::` が成功すること
- 既存の CLI 仕様が変更されないこと

### リスク・留意点
- テーブル定義の可読性を担保する設計が必要
