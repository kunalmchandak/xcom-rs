# centralize-cli-errors Proposal

## 背景
エラーレスポンスの生成（`Envelope` と `ErrorDetails`）が複数箇所に分散し、重複コードが増えています。変更時の修正漏れや一貫性の崩れが起こりやすい状態です。

## 目的
エラーレスポンスの生成を共通化し、メタ情報や終了コードの整合性を保ちながら保守性を向上させます。

## スコープ
- 共通のエラーレスポンス生成ヘルパーを追加
- 既存のエラーメッセージ・出力形式・終了コードは維持

## 非スコープ
- 新しいエラー種別の追加
- エラー内容の仕様変更

## 成果物
- エラー生成ヘルパー（例: `response` / `errors` モジュール）
- 重複コードの削減

## リスク/留意点
- 既存の分岐条件が取りこぼされる可能性
