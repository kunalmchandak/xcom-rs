# 設計メモ

## 主要な判断
### 統合テストの実行バイナリ参照
- `CARGO_BIN_EXE_xcom-rs` を使用して実行パスを取得する。
- これにより OS ごとの拡張子差分（Windows の `.exe`）や `target` 配下の構成差分を吸収できる。

### セキュリティ監査の扱い
- 監査ツールの外部依存（ネットワーク/JSON取得失敗）で CI 全体が失敗しないようにする。
- 監査の結果が「脆弱性検出」である場合は引き続き失敗させるが、外部要因の「ツール失敗」は警告として扱う。

## トレードオフ
- 監査ジョブの失敗を緩和すると、外部要因の検知は警告扱いになる。
- その代わり、CI の主目的であるビルド/テストの安定性を優先できる。
