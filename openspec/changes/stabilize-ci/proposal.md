# 変更提案: CI失敗の安定化

## 背景
- GitHub Actions の CI が複数ジョブで失敗している。
- 失敗要因が混在しており、特にクロスプラットフォームでのテスト実行とセキュリティ監査の安定性が低い。

## 目的
- CI が環境差分に左右されずに実行できるようにする。
- 監査ジョブの一時的な外部要因で、CI 全体が不必要に失敗しないようにする。

## スコープ
- 統合テストにおける実行バイナリ参照のクロスプラットフォーム化。
- セキュリティ監査ジョブの失敗条件の見直し（外部要因による失敗の緩和）。

## 非スコープ
- 監査ツールそのものの置き換え。
- プロダクション向けのビルド/リリースプロセスの変更。

## 期待する成果
- Windows を含む OS で統合テストが安定して実行できる。
- 監査ジョブが外部要因で失敗しても、CI 全体が不要に失敗しない。
