# CI安定化

## MODIFIED Requirements

### Requirement: 統合テストのバイナリ解決はプラットフォーム非依存であること

統合テストは Cargo が提供する実行バイナリ参照を使い、OS 固有の拡張子やパス差分に依存しないことを MUST とする。

#### Scenario: Windows を含む環境で統合テストが実行される
- テストは OS 固有の拡張子に依存せず実行バイナリを解決できる。
- `target/release` 固定パスは使用しない。

### Requirement: セキュリティ監査の外部要因失敗はCI全体を失敗させない

監査ツールが外部要因で失敗した場合は警告として扱い、CI の主要成果（ビルド・テスト）を阻害しないことを MUST とする。

#### Scenario: 監査ツールが外部要因で JSON 取得に失敗する
- 監査ジョブは警告として扱われ、ビルド/テスト結果には影響しない。
- 監査結果が脆弱性検出の場合は引き続き失敗とする。
