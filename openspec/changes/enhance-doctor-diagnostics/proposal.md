# enhance-doctor-diagnostics 提案

## なぜ
- doctorはローカル状態中心で、疎通/権限不足の切り分けが難しい。
- 運用時の失敗原因が「設定」なのか「権限」なのか「ネットワーク」なのか即時判断できない。

## 何を変えるか
- doctor出力に「スコープ適合性」「API疎通（任意）」の診断を追加する。
- 疎通は明示的に `--probe` 指定時のみ実行し、未実行は `skipped` として返す。
- 失敗時は `warnings` と `nextSteps` を充実させる。

### 対象範囲
- doctor診断モデルとハンドラ
- doctorのCLIオプション（`--probe`）
- doctorのテストと例

### 非対象
- 実際の投稿/検索コマンドの挙動変更
- 本番APIに依存する長時間検証

### 成果物
- `doctor`の新しい診断フィールドと明確な失敗ガイダンス
- 疎通診断のモック/スタブ付きテスト

### 受け入れ条件
- `doctor`がスコープ不足と疎通状況を返す
- `--probe`未指定時は外部アクセスを行わない

### リスク・留意点
- 疎通チェックは外部依存のため、テストではモックを必須とする
